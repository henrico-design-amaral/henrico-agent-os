# Henrico Agent OS — Foundation Brief

## Tese central

Não usar IA como atalho.

Usar IA como infraestrutura de trabalho.

O objetivo do Henrico Agent OS é transformar ChatGPT, Codex local, Codex no Antigravity, GitHub, Markdown e ferramentas locais em um sistema operacional de execução, documentação, revisão e evolução de projetos.

## Problema

O uso de IA em projetos tende a virar caos quando depende apenas de chats longos, prompts soltos e memória informal.

Os principais riscos são:

- perda de contexto
- decisões espalhadas
- retrabalho
- prompts repetidos
- escopo mal controlado
- commits misturados
- IA alterando arquivos sem plano
- dificuldade de continuar projetos em ferramentas diferentes

## Princípio operacional

Nenhuma IA é a fonte da verdade.

A fonte da verdade é o contexto versionável em Markdown.

IA lê, interpreta, executa, revisa e documenta.
Henrico pensa, decide e assina.

## Objetivo da Fase 1

Criar um orquestrador documental mínimo para manter ChatGPT, Codex local, Codex no Antigravity e o usuário operando com o mesmo contexto.

A Fase 1 não inclui:

- multiagentes complexos
- Paperclip
- Obsidian CLI
- RAG
- automações pesadas
- SaaS
- backend
- autenticação
- banco vetorial

A Fase 1 inclui:

- governança
- handoff entre ferramentas
- controle de tarefas
- decisões registradas
- quality gates
- tool map
- prompts bootstrap
- fluxo Git limpo
- documentação viva

## Frase central

Produtividade não é fazer mais coisas.

Produtividade é reduzir atrito entre intenção, decisão, execução e publicação.
